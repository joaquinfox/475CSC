Best practices associated with setting and declaring the properties of an Android user interface:

Best practices in android application development address issues or responsivenss, accessability and internationalization (Deitel & Deitel. 2017). Responsiveness is a key imperative and design principal in mobile software development. Best practices in achieveing responsive views include,

  • using a responsive layout such as ConstraintLayout
  • using different layouts for different devices.
Layouts are xml files in directory res/layout. They can be written or built with the LayoutEditor, so that each activity is paired to a particular layout. For example, MainActivity.java is paired to activity_main.xml by invoking setContentView(R.layout.activity_main);

Best practices call for accessible apps so that "users of diverse abilities to navigate, understand, and use your UI" (material.io. Accesibility)

Best practices for building an accessible UI include:
  • Structuring components semantically and with good text descriptors that screen readers such as TalkBack can read. This is generally done by setting the contentDescription attribute of a view.
  • Providing feedback through visual and textual cues, to guide user through an app.
  • Provide a clear navigation interface.
  • Clearly visible elements.
  • Sufficient contrast and size.
  • A clear hierarchy of importance.
  • Highlight key information.

Much as we can specify different layouts for different devices we can also specify different layouts for different countries. Android can then automatically load the resources that match the devices locale settings (Deitel & Deitel. p. 70).
